case class Entropy[P <: Position, S <: Position with ExpandablePosition, W](count: Extract[P, W, Double], filter: Boolean = true, strict: Boolean = true, nan: Boolean = false, negate: Boolean = false, log: (Double) â‡’ Double = ...) extends AggregatorWithValue[P, S, S] with FilterAggregator[P] with Product with Serializable

Compute entropy.

count

Object that will extract, for cell, its corresponding count.

filter

Indicates if only numerical types should be aggregated. Is set then all categorical values are filtered prior to aggregation.

strict

Indicates if strict data handling is required. If so then any non-numeric value fails the reduction. If not then non-numeric values are silently ignored.

nan

Indicator if 'NaN' string should be output if the reduction failed (for example due to non-numeric data).

negate

Indicator if negative entropy should be returned.

log

The log function to use.

    Note

    An Option is used in the return type to allow aggregators to be selective in what content they apply to. For example, computing the mean is undefined for categorical variables. The aggregator now has the option to return None. This in turn permits an external API, for simple cases, where the user need not know about the types of variables of their data.
